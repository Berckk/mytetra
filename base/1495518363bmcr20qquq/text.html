Разделяй и тестируй.

Разделить интерфейс на части, тестировать каждую часть изолировано.

Всю цепочку тестировать всего несколькими тестами. Можно и вручную, если не доверяем автоматизированным тестам.

Тесты должны быть атомарными, независимыми.

Вместо "тяжелых" компонентов использовать mocks.

* Например, чтобы запустить приложение на web-сервере можно на тесте запускать на легких серверах (Jetty, Play!).

* Вместо боевой БД можно использовать in-memory базы данных (SQLLigth, H2).

* Наполнение in-memory базы данных с помощью инструмента LiqueBase

* Вместо внешний сервисов использовать свой эмулятор.
